# Type System & Specification Details

Цей модуль присвячений **системі типів JavaScript** та алгоритмам специфікації ECMA-262, які керують порівнянням і приведенням типів (Type Coercion).

Розуміння цих механізмів дозволяє писати передбачуваний код, уникати класичних помилок з "дивною" поведінкою операторів та розуміти, як рушій V8 оптимізує операції порівняння.

---

## 🎯 Що ви вивчите в цьому модулі?

1. **Алгоритми порівняння:** Як саме працює рекурсивний алгоритм `==` (Abstract Equality) та чому `===` (Strict Equality) працює набагато швидше на рівні байткоду.
2. **Abstract Operations:** Механіку внутрішніх операцій `ToPrimitive`, `ToNumber`, `ToString`, які викликаються рушієм неявно.
3. **Винятки специфікації:** Чому об'єкти `Date` поводяться не так, як звичайні об'єкти під час додавання (`+`).
4. **V8 Optimizations:** Вплив зміни типів на Inline Caches (IC) та деоптимізацію коду.

---

## 📂 Розділи модуля

### 1. [Abstract Equality vs Strict Equality](./01-abstract-vs-strict-equality/README.md)
*   **Про що:** Глибокий розбір алгоритму порівняння ЕСМА-262.
*   **Ключові концепції:** Каскадна конверсія типів, рекурсивні виклики `ToNumber` та `ToPrimitive`, парадокс `[] == ![]`.
*   **Візуалізація:** Інтерактивний покроковий симулятор бінарного порівняння.

### 2. [Primitive Coercion Algorithms](./02-primitive-coercion/README.md)
*   **Про що:** Детальний розбір внутрішніх операцій приведення об'єктів та примітивів.
*   **Ключові концепції:** `valueOf()` vs `toString()`, різниця в `hint` (default, number, string), винятки для об'єктів `Date`, суворість `ToNumber` порівняно з `parseInt`.
*   **Візуалізація:** Симулятор роботи `ToPrimitive` для `Object` та `Date`.

---

## 🚀 Навіщо це Senior / Middle розробнику?

*   **Уникнення Memory Flooding:** Розуміння, як неявне приведення до рядка в циклах створює дублікати в `StringTable` рушія V8.
*   **Захист від Edge Cases:** Впевненість при роботі з користувацьким вводом (де часто приходять порожні рядки `""` або `null`).
*   **Продуктивність:** Знання того, як тримати call-site мономорфним для Inline Caches у V8, уникаючи Cast-операцій.
